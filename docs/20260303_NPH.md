# 2026-03-03 NPH Protocol

## 1. 文档范围

这份文档描述的是**当前仓库里已经实现的 NPH protocol**，不是泛化的教材版 NPH 笔记。
它对应的主要代码入口是：

- `MDSimulation::step_single_NPH(...)`
- `tests/run_NPH_test/run_test_NPH.cpp`
- `run/series_NPH.cpp`

当前实现已经完全切换到：

- 二维体系
- 周期性盒子
- 面积涨落的 barostat

不再使用旧的 wall-piston 模型。

## 2. 当前 NPH 的物理语义

### 2.1 这是二维 NPH，不是三维 NPH

当前代码工作的几何对象不是三维体积 `V`，而是二维面积：

```text
A = Lx * Ly
```

因此这里的 NPH 更准确地说是：

> 2D periodic-box extended-system NPH

系统盒子由两个全局长度描述：

- `Lx`
- `Ly`

### 2.2 边界条件

当前 NPH 在两个方向都是周期性边界：

- `x` 方向周期性
- `y` 方向周期性

所以现在不存在：

- 反射墙
- 顶部移动活塞
- `y` 方向的 wall pressure

现在的 `Ly` 变化只是盒子尺寸变化，不是上壁面在移动。

### 2.3 盒子形状保持不变

当前 barostat 不是各向异性的，而是**固定纵横比**的面积 barostat。

初始化时定义：

```text
gamma = Lx / Ly
```

随后 `gamma` 保持不变。
因此面积变化时，`Lx` 和 `Ly` 只是按同一个线性缩放因子变化：

```text
lambda = sqrt(A_new / A_old)
Lx_new = lambda * Lx_old
Ly_new = lambda * Ly_old
```

只有在触发 `Ly` 的上下限保护时，才会先 clamp `Ly`，再由固定的 `gamma` 反推出 `Lx`。

## 3. 当前实现维护的动力学变量

形式推导里常见的是：

- 缩放坐标 `s_i`
- 共轭动量 `p_{s_i}`
- 体积 `V`
- 活塞动量 `p_V`

但当前代码没有显式存这套变量。
当前实现真正演化的是：

- 粒子真实坐标 `r_i = (x_i, y_i)`
- 粒子真实速度 `v_i = (vx_i, vy_i)`
- 粒子加速度 `a_i`
- 盒子长度 `Lx, Ly`
- 面积动量 `p_A`
- 固定纵横比 `gamma`

其中 barostat 质量为：

```text
W = barostat_mass
```

代码里可直接读出的面积变化率是：

```text
dA/dt = p_A / W
```

初始面积动量由配置给出：

```text
p_A(0) = W * barostat_area_rate_init
```

## 4. NPH 使用的瞬时压力

当前代码里的瞬时压力是二维标量 virial pressure：

```text
P_inst = [ sum_i m_i |v_i|^2 + sum_{i<j} r_ij . F_ij ] / (2A)
```

其中：

- `A = Lx * Ly`
- `r_ij` 使用 minimum-image 位移
- minimum-image 在 `x`、`y` 两个方向都生效

barostat 的驱动力是：

```text
dp_A/dt = P_inst - P_ext
```

其中 `P_ext` 就是 `step_single_NPH(P_ext)` 传进去的外压目标值。

## 5. 单步 NPH 积分器

下面描述一次 `step_single_NPH(P_ext)` 实际做了什么。

### 5.1 惰性初始化

第一次进入 NPH 步进时，代码会做一次初始化：

1. 读取当前 `Lx`、`Ly`
2. 记录固定纵横比

```text
gamma = Lx / Ly
```

3. 初始化面积动量

```text
p_A = W * barostat_area_rate_init
```

这个初始化只发生一次。

### 5.2 计算旧状态压力

在每个 NPH 时间步开始时，先计算当前状态下的瞬时压力：

```text
P_old = P_inst(current positions, current velocities, current box)
```

这里的压力计算已经假定 `x`、`y` 都是周期性边界。

### 5.3 粒子速度半步 kick

代码先用上一步已经存好的加速度做半步更新：

```text
v <- v + (dt/2) a
```

这对应 velocity-Verlet 的第一半步。

### 5.4 面积动量半步 kick

然后更新 barostat 的面积动量：

```text
p_A <- p_A + (dt/2) * (P_old - P_ext)
```

这对应 barostat 的第一半步。

### 5.5 面积全步更新

用更新后的面积动量推进面积：

```text
A_new = A_old + dt * (p_A / W)
```

如果 `A_new <= 0` 或者不是有限数，程序会直接 abort。

### 5.6 在固定纵横比下恢复新盒子

因为 `gamma` 固定，所以：

```text
Ly_new = sqrt(A_new / gamma)
Lx_new = gamma * Ly_new
```

这说明当前 NPH 只允许面积涨落，不允许盒子形状独立变化。

### 5.7 `Ly` 的上下限保护

代码对 `Ly` 加了一个数值安全范围：

```text
Ly_min = max(1e-12, barostat_height_min_ratio * Ly0)
Ly_max = max(Ly_min, barostat_height_max_ratio * Ly0)
```

这里的 `Ly0` 是构建 simulation environment 时记下来的初始盒高。

如果 `Ly_new` 超出这个区间，就会被截断到边界值。
然后再用固定纵横比重新计算 `Lx_new`。

要注意：

> 这个 clamp 只是数值保护，不代表真实的物理墙。

### 5.8 scale-and-drift

接下来定义线性缩放因子：

```text
lambda = sqrt(A_new / A_old)
```

然后对每个粒子做：

```text
v <- v / lambda
r <- lambda * r + dt * v
```

这里有两个动作：

- 先按盒子缩放修正速度
- 再在新盒子上做一次漂移

随后粒子坐标会被 wrap 回新盒子：

```text
x <- wrap(x, Lx_new)
y <- wrap(y, Ly_new)
```

所以这一步本身已经明确说明当前 NPH 是双周期边界。

### 5.9 提交新盒子并重建 MPI 几何

坐标更新后，代码会：

1. 写回新的 `Lx_new`、`Ly_new`
2. 重新计算每个 rank 的 `x_min`、`x_max`
3. 按新的 `Lx` 做粒子迁移
4. 重建 halo 粒子

当前并行分解仍然是沿 `x` 方向做的。

### 5.10 重算力

盒子和粒子状态都更新完之后，代码调用一次新的 force 计算，得到更新后状态的加速度。

这一步是后半个 velocity-Verlet 所需要的。

### 5.11 计算新状态压力

然后再次计算压力：

```text
P_new = P_inst(updated state)
```

这同样使用 `x`、`y` 双周期的 virial 压力定义。

### 5.12 粒子速度第二个半步

粒子速度后半步更新为：

```text
v <- v + (dt/2) a_new
```

### 5.13 面积动量第二个半步

barostat 的面积动量再做一次半步更新：

```text
p_A <- p_A + (dt/2) * (P_new - P_ext)
```

### 5.14 clamp 后的动量处理

如果这一步前面已经触发了 `Ly` 的 clamp，而且更新后的 `p_A` 还想继续把盒子往同一个越界方向推，代码会把：

```text
p_A = 0
```

这样做的目的是防止 barostat 一直朝着被 clamp 的方向累积动量。

### 5.15 更新时间

最后：

```text
t <- t + dt
```

并把 `last_instant_pressure` 存成 `P_new`。

## 6. 它和形式推导的差别

设计讨论里常用的形式推导是：

- 用缩放坐标 `s_i`
- 用共轭动量 `p_{s_i}`
- 用体积 `V`
- 用活塞动量 `p_V`

当前代码实现是一个更偏工程实现的等价版本：

- 直接存真实坐标和真实速度
- 在二维里用面积 `A`
- 保持盒子纵横比不变
- 直接在真实空间做盒子缩放和粒子 rescale

因此，当前实现更准确的描述是：

> 一个二维、固定纵横比、双周期边界、基于面积自由度的 extended-system NPH barostat

而不是字面上把 `(s_i, p_{s_i}, V, p_V)` 整套 canonical 变量全部显式存起来。

## 7. Driver 层的 protocol

仓库里的 NPH 工作流通常不是裸跑 `step_single_NPH(...)`，而是放在一个三阶段协议里：

- `AUTO_TARGET`
- `NPH_WARMUP`
- `NPH_PRODUCTION`

这套状态机逻辑目前出现在：

- `tests/run_NPH_test/run_test_NPH.cpp`
- `run/series_NPH.cpp`

## 8. Phase 1: `AUTO_TARGET`

### 8.1 什么时候进入这个阶段

如果配置里：

```text
pressure_target_mode == "auto_initial"
```

那么运行一开始会进入 `AUTO_TARGET`。

否则会直接跳过 `AUTO_TARGET`，进入 `NPH_WARMUP`，并直接使用配置文件里的 `P_target`。

当前实现要特别注意一点：

> 代码只检查是否精确等于 `"auto_initial"`。
> 任何别的字符串都会被当成“不要自动估计压力目标”。

### 8.2 `AUTO_TARGET` 阶段实际做什么

在这个阶段，driver 不跑 NPH。
它做的是：

```text
step_single_nose_hoover()
```

然后测量一次瞬时压力。

所以这一阶段本质上是：

> 用固定盒子的 Nose-Hoover 预热体系，并从这个预热轨道里估计一个合理的目标压力。

### 8.3 `P_target` 怎么得到

每隔 `pressure_eval_interval_steps`，driver 会把当前压力塞进 `auto_samples`。

当 `AUTO_TARGET` 总步数达到：

```text
pressure_target_auto_steps
```

时，取平均：

```text
P_target = mean(auto_samples)
```

然后切换到 `NPH_WARMUP`。

## 9. Phase 2: `NPH_WARMUP`

进入这个阶段后，每一步都调用：

```text
step_single_NPH(P_target)
```

这个阶段的目的不是采样生产数据，而是让 barostat 收敛到目标压力附近。

### 9.1 滚动窗口监控

每隔 `pressure_eval_interval_steps`，当前压力会被压进一个滚动窗口。
窗口最大长度是：

```text
pressure_window_samples
```

driver 基于这个窗口计算：

- rolling mean
- rolling standard deviation

### 9.2 warmup 完成判据

至少先跑满：

```text
nph_warmup_min_steps
```

之后，如果滚动窗口已经装满，就会检查：

```text
rel_err = |mean(P_window) - P_target| / max(1e-12, |P_target|)
rel_std = std(P_window) / max(1e-12, |P_target|)
```

如果二者都满足：

```text
<= pressure_tolerance_rel
```

并且连续稳定窗口数达到：

```text
pressure_stable_required_windows
```

就会切换到 `NPH_PRODUCTION`。

### 9.3 强制结束 warmup

如果到了：

```text
nph_warmup_max_steps
```

还没有满足稳定判据，driver 也会强制切换到 `NPH_PRODUCTION`。

所以 `nph_warmup_max_steps` 是 warmup 的硬上限。

## 10. Phase 3: `NPH_PRODUCTION`

这个阶段的积分器并没有换，还是：

```text
step_single_NPH(P_target)
```

只是语义变了：

- 现在把体系视为已经足够接近平衡
- 这一段轨道当作正式生产数据

当生产阶段累计步数达到：

```text
nph_steps
```

时，运行结束。

## 11. MPI 下的控制逻辑

这套 protocol 的状态决策由 rank 0 负责。

每个外层循环里大致是：

1. 所有 rank 推进一步局部动力学
2. rank 0 更新计数器并决定是否换 phase
3. rank 0 广播：
   - `phase`
   - `P_target`
   - `finished`

这样整个 MPI 作业会始终处于同一个 phase。

### 12.3 主要输出量的含义

输出里常见字段包括：

- `P_inst`
  当前瞬时压力

- `P_target`
  当前外压目标

- `Lx`, `Ly`
  当前盒子长度

- `area`
  当前盒子面积

- `dA_dt`
  当前面积变化率，也就是 `p_A / W`

- `a_barostat`
  一个便于监控的 barostat 加速度量：

```text
a_barostat = (P_inst - P_target) / W
```

- `rolling_mean`, `rolling_std`
  warmup 阶段的压力稳定性诊断量

## 13. 直接控制 NPH 的配置项

当前最关键的参数是：

- `barostat_mass`

  控制面积自由度的惯性。
  越大，面积变化越慢；越小，面积响应越快。

- `barostat_area_rate_init`

  初始 `dA/dt`。
  因为 `p_A = W * dA/dt`，所以这等价于设置初始 barostat 动量。

- `barostat_height_min_ratio`

  `Ly` 的下边界，相对于初始 `Ly0` 的比例。

- `barostat_height_max_ratio`

  `Ly` 的上边界，相对于初始 `Ly0` 的比例。

- `pressure_target_mode`

  如果是 `"auto_initial"`，则先做一段固定盒子的压力自估计；
  否则直接使用配置里的 `P_target`。

- `P_target`

  外压目标，也就是 `step_single_NPH(P_target)` 里的 `P_ext`。

- `pressure_target_auto_steps`

  `AUTO_TARGET` 阶段的长度。

- `pressure_eval_interval_steps`

  压力评估和 rolling window 更新的间隔。

- `pressure_window_samples`

  rolling window 的样本数。

- `pressure_tolerance_rel`

  同时用于约束压力均值误差和压力波动强度的相对容差。

- `pressure_stable_required_windows`

  连续稳定窗口数阈值。

- `nph_warmup_min_steps`

  允许进入 production 之前，warmup 至少要跑多少步。

- `nph_warmup_max_steps`

  warmup 最多跑多少步，超过后强制切入 production。

- `nph_steps`

  production 阶段总共要跑多少步。

## 14. 如何理解 `barostat_mass`

`barostat_mass` 的作用是把盒子自由度和粒子自由度的时间尺度分开。

如果它太小，通常会出现：

- 面积震荡太激烈
- 压力目标被频繁过冲
- 盒子容易反复撞到 `Ly` 的 clamp

如果它太大，通常会出现：

- 体系往目标压力收敛很慢
- warmup 拉得很长
- 如果 warmup 参数又给得太短，production 可能在盒子还没充分适配前就开始

## 15. 当前实现的几个重要限制

### 15.1 它是二维面积 barostat

不是三维体积 barostat。

### 15.2 它固定纵横比

`Lx/Ly` 在 NPH 初始化之后不再独立变化。
所以这不是一个可分别调节 `x` 和 `y` 应变的各向异性压强控制器。

### 15.3 它是双周期边界

当前 NPH 路径没有任何 wall physics。
`barostat_height_min_ratio` 和 `barostat_height_max_ratio` 只是数值安全边界，不是物理墙位置。

### 15.4 压力和力是分开归约的

当前实现里：

- 压力通过专门的 virial reduction 计算
- 粒子加速度来自 force kernel 存下来的结果

因此每个 NPH 步里会出现：

- 一次旧态压力评估
- 一次 scale/drift 后的 force rebuild
- 一次新态压力评估

### 15.5 当 `P_target` 接近 0 时，warmup 判据会变得很严格

因为归一化分母用的是：

```text
max(1e-12, |P_target|)
```

所以如果手动指定的 `P_target` 非常接近 0，rolling window 的相对误差和相对波动可能会显得异常大。

## 16. 一句话总结

当前仓库里的 NPH protocol 可以概括成：

> 先自动估计或直接指定目标压力，再在一个二维、双周期、固定纵横比的涨落盒子里运行面积 barostat；等滚动压力统计满足稳定条件后，把同样的 NPH 动力学继续用于 production。
